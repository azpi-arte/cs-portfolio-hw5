User-Level Threads & Synchronization – UCSD CSE120

Description

Implemented a full user-level threading system for an operating systems course, covering context switching, cooperative scheduling, mutual exclusion, and semaphore-based synchronization. The work centered around extending a constrained starter kernel and producing a complete threading runtime in C.

Role

Developed the thread manager, per-thread stacks, FIFO scheduling, and semaphore primitives. Designed test cases to validate correctness under contention, interleavings, and edge-case yield paths.

    Tech Utilized

  • C
  • User-level threading
  • Context switching (setjmp/longjmp)
  • Semaphores
  • Scheduling
  • Low-level stack management

    Challenges

  1. Context Control

    Used setjmp/longjmp to implement manual context switching while maintaining valid call stacks and activation records across thread boundaries.

  2. Stack Layout

    Constructed isolated stacks for each thread without dynamic allocation, using fixed-size buffers and careful placement of frame-dependent parameters.

  3. Thread Lifecycle

    Generalized thread creation and teardown beyond the two-thread scaffold, supporting reuse of IDs and maintaining a FIFO ready queue.

  4. Yield Semantics

    Implemented yields between arbitrary threads with correct return propagation, enabling cooperative multitasking and reproducible control transfer.

  5. Semaphores & Mutual Exclusion

    Built semaphore operations atop the scheduler to block, queue, and wake threads while guaranteeing atomicity in critical sections.
